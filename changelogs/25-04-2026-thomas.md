## Fixed
- API key validation now correctly rejects gibberish keys for all providers.
  - Anthropic: validated via native `x-api-key` header against the Anthropic models endpoint.
  - Gemini: validated via the Google generative language models endpoint.
  - OpenAI-compatible providers (OpenAI, Groq, DeepSeek, OpenRouter, custom): tightened exception handling — `401`/`403` responses now always reject rather than silently passing.
  - Empty keys are rejected for all providers except Ollama and localhost endpoints.
- Structured response parsing now handles providers (e.g. Gemini) that serialize `blocks` array elements as JSON strings instead of nested objects, preventing fallback rendering on valid responses.
- Suppressed noisy Gemini SDK warning about non-text parts in multi-part responses — the tool call was always processed correctly; the warning was cosmetic.

## Refactored
- Extracted agent invocation, streaming, and animation logic from `main.py` into a new dedicated `runner.py` module.
  - `runner.py` now owns: `STATUS_LABELS`, `TOOL_MESSAGES`, `_tool_message`, `_block_key`, `_animate_status`, `_invoke_agent_stream`, `_extract_response_from_state`, and the public `run_agent` entry point.
  - `main.py` is reduced to ~115 lines covering only CLI startup, help display, and the command loop.
  - `_run_agent` renamed to `run_agent` (public) to reflect its cross-module role.
