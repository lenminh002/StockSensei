## Added
- High-quality terminal visual charting using `plotext` in `renderers.py`:
  - `LineChartBlock` rendered with `plotext.plot` for historical closing price trends.
  - `BarChartBlock` rendered with `plotext.bar` for volume and vertical column comparisons.
  - `CandlestickChartBlock` rendered with `plotext.candlestick` for stock price OHLC trends.
- New stock research agent tools in `tools.py`:
  - `build_time_comparison_line_visual` to draw comparison line trends of returns (percent) or prices across multiple tickers over a time period.
  - `build_line_chart_visual` to render a single stock close-price trend.
  - `build_volume_chart_visual` to render a single stock trading volume column chart.
  - Upgraded `build_history_chart_visual` to candlestick charts and `build_price_chart_visual` to auto-fallback to volume column charts for single tickers.
  - Added natural-language period parsing utility `normalize_period` for yfinance (e.g. "5 days" -> "5d", "3 months" -> "3mo").

## Refactored
- Decoupled StockSensei Core service from the Terminal UI runner layer:
  - Extracted core progress labels, API error classifications, and agent state parsing out of `runner.py` into `stocksensei/core/service.py`.
  - Removed all obsolete direct agent execution loops (`run_agent`, `_invoke_agent_stream`) from the terminal UI runner module.
- Migrated monolith modules to targeted core/UI packages:
  - Migrated `ui_blocks.py` into focused modules: `block_models.py`, `block_formatting.py`, `responses.py`, `block_builders.py`, and fallback `block_renderer.py`.
  - Moved `agent.py` to `stocksensei/core/agent.py`.
  - Moved `tools.py` to `stocksensei/core/tools.py`.
  - Split `config.py` into core config persistence (`stocksensei/core/config.py`) and terminal CLI interactive selection flows (`stocksensei/ui/terminal/config.py`).
  - Moved REPL command prompt loop to `stocksensei/ui/terminal/command_prompt.py`.
- Created root backward-compatibility facades re-exporting all public symbols:
  - Root facade modules created for `ui_blocks.py`, `agent.py`, `tools.py`, `config.py`, `providers.py`, and `command_prompt.py` to keep legacy imports working.

## Fixed & Cleaned Up
- Removed `SparklineBlock` and its ASCII chart fallback renderings:
  - Completely deleted the `SparklineBlock` model schema, builder, and registry mapping.
  - Replaced the ASCII candlestick/line/bar fallback drawings in `block_renderer.py` with structured, red-bordered error panels explaining graphical plotting limitations.
  - Cleaned up obsolete ASCII bar and candlestick drawing helpers from `block_formatting.py`.
- Decoupled extension loader imports in `discovery.py` and `manager.py` to target core config directly instead of root facade modules, resolving a startup `ImportError`.
- Cleaned up unused variables and imports, including the unused `asyncio` import in `service.py` and leftover `SparklineBlock` imports in `renderers.py`.
