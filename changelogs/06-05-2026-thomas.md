## Fixed
- Fixed "Event loop is closed" crash when using Gemini on the second prompt and beyond.
  - Root cause: `asyncio.run()` was called once per prompt, creating and destroying a fresh event loop each time. Gemini's `httpx` connection pool caches connections bound to the old loop; on the next prompt it tries to close those stale connections, reaches for the destroyed loop, and raises `RuntimeError: Event loop is closed`.
  - Fix: replaced per-prompt `asyncio.run()` with a process-lifetime event loop via a new `_get_loop()` helper in `runner.py`. The loop is created once on first use and reused for every subsequent prompt, keeping Gemini's (and any other provider's) pooled connections bound to a loop that stays alive for the whole session.
  - Added full traceback logging to the `except Exception` fallback in `run_agent()` so future unexpected errors surface with a complete stack trace instead of just a one-line message.
