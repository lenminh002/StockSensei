## Added
- Added extensible StockSensei Core foundation for future Web UI and plugins.
  - Introduced `stocksensei/` package with core services, registries, extension support, and terminal UI seams while preserving the current `main.py` entry point.
  - Added `StockSenseiService` with typed async events (`status`, `tool_start`, `tool_end`, `block`, `final`, `error`) and a core session model.
  - Added tool, Visual Block, and command registries; built-in tools and blocks now seed those registries.
  - Added trusted Python extension discovery and activation API with required extension ids/API versions, lifecycle hooks, diagnostics, and project/global/configured/entry-point sources.
  - Added terminal renderer seam and fallback rendering for custom Visual Blocks.
  - Added `/extensions` diagnostics command.
  - Documented domain terms and ADRs for trusted extensions and core extension architecture.

## Changed
- Routed the CLI through `StockSenseiService` so Terminal UI consumes the same core event stream planned for the future Web UI.
- Migrated config writes to `~/.config/stocksensei/config.json` while keeping legacy `~/.stocksensei_config.json` read fallback.
- Improved startup prompts for nushell compatibility by avoiding Rich prompts for provider setup input.
